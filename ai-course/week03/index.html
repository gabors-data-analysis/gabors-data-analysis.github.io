Week 03: Reporting your data analysis

Creating inputs and organizing a short data analytics report with AI

About the class

This week turns your Week 2 documentation skills into a full mini-report workflow. You will move from an empirical question to exhibits and a short written conclusion, while using AI as an assistant for planning, coding, and critique.

Objectives

By the end of class, students will be able to:

  • Translate a broad research topic into a precise country-level question using documented variables.
  • Design and produce a short report with two graphs, one regression, and a short conclusion.
  • Compare one-shot AI output with iterative AI workflows and explain quality differences.
  • Detect half-truths and over-claims in AI-generated analysis, especially causal language mistakes.

Preparation BEFORE class

Required reading and setup (before class)

  1. Read Bekes-Kezdi (2021), Chapters 3-4 and 7-10.
  2. Download and open these files:
    • WVS_GDP_merged_data.csv
    • README_WVS.md
    • codebook.pdf
  3. Review the World Values Survey case study page.
  4. Ensure your coding environment is ready (VSCode + Copilot or an AI CLI tool).

The data

  • This is aggregated data: country level
  • Year: Wave 7 of the WVS – survey was conducted at different years.
  • Combined with World Bank data: at year when survey was conducted
  • GDP: level USD, level USD PPP, level USD PPP per capita.
  • population

Class plan

🤝 Part 0: Calibration and quality bar (15 min)

Quick review

  • Q+A

  • Common AI errors in documentation and reporting

  • Reminder: AI is an input tool, not an automatic final writer

  • What “good enough” versus “submission quality” means### How does a good report look like?

    • How to write a good short report: structure
    • good graphs and tables
      • Make sure precise language. Recap on causal language.

Problems with AI generated reports.

Use AI as input (like advanced google search) not as output writer, because

  • Creates “average” / generic / bland / repetitive text

  • Not your style and not your exact plan

  • Convincing but would have errors especially with info outside of context window

  • not precise enough, especially using causal language inappropriately.

  • Too broad (like adds further research)

Target output for today

By end of class each group should have:

  1. A one-page report skeleton with section headings.
  2. A clear exhibit plan (Graph 1, Graph 2, Regression Table 1).
  3. One polished coefficient interpretation in non-causal language.

📚 Part 1: Data grounding and variable mapping (20 min)

Group task

Form groups of 2-3 students. Pick one question:

  • Is there a relationship between income level and trust?
  • Is there a relationship between income level and happiness?
  • Is there a relationship between income level and gender attitudes?

Build a mini analysis spec (no coding yet)

Each group writes five lines:

  1. Question statement (association wording).
  2. Chosen y variable(s) from codebook (Q columns).
  3. Chosen x income variable (GDP_USD or PPP variant; transform if needed).
  4. Planned heterogeneity split using Population.
  5. One sentence on expected sign and one sentence on limits.

🧭 Part 2: No-AI report design sprint (15 min)

Design before generation

Without AI drafting text, groups define the report structure:

  1. Introduction (2-3 sentences)
  2. Data and variables
  3. Results with exhibits
  4. Conclusion (80-100 words)

Required exhibits

  • Graph 1: Main income-outcome pattern
  • Graph 2: Heterogeneity by country size groups (population-based split)
  • Table 1: Regression with one precisely interpreted coefficient

Guardrail checklist

  • Is every variable defined from the documentation?
  • Is the graph title aligned with non-causal language?
  • Is there at least one explicit data limitation?

🤖 Part 3: AI round A - one-shot attempt (15 min)

Task

Use one prompt to generate the full report draft from your spec.

Capture these artifacts

  1. The exact one-shot prompt.
  2. Output draft (text and requested exhibits).
  3. Three quality issues found by your group.

Focus

Look for plausible but incorrect details, vague variable definitions, and accidental causal claims.

🔁 Part 4: AI round B - iterative workflow (30 min)

Task sequence

Run targeted prompts in order:

  1. Validate variable definitions against codebook.pdf.
  2. Generate Graph 1 exactly as specified.
  3. Generate Graph 2 with your chosen population grouping rule.
  4. Run regression and draft one coefficient interpretation.
  5. Rewrite interpretation to remove causal overreach.

Verification requirements

Before accepting output, check:

  • Variable names match the dataset exactly.
  • The transformation (for example, log GDP per capita PPP) is stated.
  • Regression interpretation includes scale and uncertainty (SE or CI).
  • Limitation statement reflects country-level aggregation.

🧩 Part 5: AI round C - extended context window (15 min)

Task

Create a reusable system prompt with:

  • writing style preferences
  • output format requirements
  • rules for causal language
  • verification checklist

Rerun the report generation and compare with rounds A and B.

Compare on three criteria

  1. Control over structure
  2. Precision of variables and interpretation
  3. Editing effort needed to reach submission quality

End-of-week discussion points

  • When did one-shot prompting fail in your workflow?
  • Which iterative step produced the largest quality gain?
  • What checks best exposed half-truth outputs?
  • Which part still needed human rewriting the most?
